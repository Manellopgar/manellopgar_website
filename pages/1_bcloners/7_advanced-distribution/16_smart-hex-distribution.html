Smart Hex Distribution

This distribution can be loaded directly using the Advanced Distribution button in the BCloners Extension Panel or place it manually if you change the Distribution Menu to Advanced and set the Distribution Object to the parameter Distribution Object. Copies are placed in a regular grid, similar to the standard Honeycomb arrangement of the BCloner object. However, this Modifier Distribution offers even more options for selecting the desired distance between the clones and additional scaling and coloring options.

Smart Hex Distribution layout example

Distribution Logic

[PARAM] Clones: Iterate

The distribution automatically runs through all objects in sequence from top to bottom, as they were grouped under the Cloner object. After placing the last object, this sequence starts again with the topmost object under the loner object.

[PARAM] Clones: Random

The object currently to be cloned is selected at random from the objects that have been grouped to the clone object.

Geometry & Orientation

[PARAM] Orientation

The calculation is based on a virtual rectangular area on which the clones are placed. This direction indicates the orientation of the perpendicular on this rectangular surface.

[PARAM] Count Width

The clones are created within a rectangular area. Here you can specify how many copies should be created horizontally (Count Width).

[PARAM] Count Height

The clones are created within a rectangular area. Here you can specify how many copies should be created vertically (Count Height).

Distribution Modes

[PARAM] Mode: Per Step

The horizontal and vertical distances between the clones can be specified using the following settings Size Width and Size Height.

[PARAM] Mode: End Point

In this mode, the distances between the clones change automatically when the Size Width or Size Height is changed.

[PARAM] Mode: Circle

The clones are treated as circles with a fixed radius. This radius can be freely selected with the following Radius parameter. The distances between the clones are automatically calculated so that there is no overlap between these circle radii. Additional distances between the clones can be configured via the Gap value.

[PARAM] Mode: Hex

This assumes that the clones are hexagonal, e.g., formed by corresponding n-Side splines or hexagonal Cylinder objects. In this case, the Radius value used on these objects can also be used directly here for the subsequent Radius value in order to obtain perfectly fitting clone placements. The separate Gap value for adjusting the clone distances is also available here.

Sizing Parameters

[PARAM] Size Width

Adjust the Size Width Between Clones.

[PARAM] Size Height

Adjust the Size Height Between Clones.

[PARAM] Radius

For the Circle and Hex Modes, the size of the clones is described by this Radius. With the Circle setting it is actually a circle radius, in Hex mode the radius of the corner points describing a hexagon is described. If you select the value here identically to the Radius specification on a Cylinder primitive object or n-Side spline, for example. The distances between the clones can also be extended by the Gap value.

[PARAM] Gap

This can be used to create an additional gap between the clones, which surrounds them like a joint.

Scaling Options

[PARAM] Match Scale To: None

The size of the cloned objects remains unchanged.

[PARAM] Match Scale To: Width/Height

The X size of the cloned objects adapts to the corresponding width/height of the subdivision area on the virtual rectangle.

[PARAM] Match Scale To: Constrained (by default)

The scaling is always based on the shortest edge of the respective subdivision area within the virtual rectangle and scales the clones proportionally to this.

[PARAM] Match Scale To: Scale To Fit

The X and Y sizes of the clones are scaled identically to the available space in the corresponding area of the virtual rectangle. This inevitably leads to a distortion of the clones.

Facing & Randomization

[PARAM] Scale Facing: None

The clones can be scaled freely, percentage based on their original size in this direction via the Scale Facing value.

[PARAM] Scale Facing: Set

The size of the clones in this direction can be specified in absolute terms using the Scale Set value.

[PARAM] Scale Facing: Proportional (by default)

The size of the clones in this direction is based on the scaling in the other axis directions, but can also be corrected as a percentage using the Scale value.

[PARAM] Scale Facing: Random

The clones are scaled randomly in the corresponding direction. However, with Random Scale Min and Random Scale Max you can define percentage maximums that refer to the original dimensions of the cloned objects in this direction.

[PARAM] Scale Set

For the Scale Facing Proportional mode, an additional multiplier is specified for scaling in a direction perpendicular to the placement plane. The scaling in the direction of width and height remains unaffected by this.

[PARAM] Facing Scale

For the Scale Facing Set mode, use this to specify the absolute dimension of the clones in a direction perpendicular to the distribution plane.

[PARAM] Min

For the Scale Facing Random mode, this defines the smallest and largest scaling in the direction perpendicular to the distribution plane. The percentage values refer to the original size of the clone bounding box in this direction.

[PARAM] Max

For the Scale Facing Random mode, this defines the smallest and largest scaling in the direction perpendicular to the distribution plane. The percentage values refer to the original size of the clone bounding box in this direction.

[PARAM] Seed

This distribution offers random modes based on this value when selecting the objects to be cloned.
