Stack Distribution

This distribution can be loaded directly using the Advanced Distribution button in the BCloners Extension Panel or placed manually if you change the Distribution Menu to Advanced and set the Distribution Object to the parameter Distribution Object. Stack Distribution is always suitable when object copies are to be displayed stacked or lined up exactly depending on their dimensions. Think of a pile of magazines or books, for example.

Stack Distribution

Clones Settings

  • Clones: Iterate – The distribution automatically runs through all objects in sequence from top to bottom, as they were grouped under the Cloner object. After placing the last object, this sequence starts again with the topmost object.
  • Clones: Random – The object currently to be cloned is selected at random from the objects that have been grouped to the clone object.

Stack Settings

  • Stack: Count – The absolute number of clones is specified via the following Count value.
  • Stack: Length – The number of clones is determined indirectly by specifying a distance to be filled by the clones; the actual number of clones depends on the following Length value and the dimensions of the clones.

Length & Count Parameters

Length: For the Length Stack mode, enter the route length to be filled by the object copies here. Please note that deviations from this specification may occur by default if the available objects cannot be placed completely in this route. A gap can then be left at the end of the specified route. If the route is to be completely filled with clones, the separate Stretch to Fill option is available. This adjusts the size of the clones so that the specified distance can be completely filled.

Count: For the Stack mode Count, enter the desired number of object copies here. The number of points to create is determined by this value.

Offset & Direction

  • Offset: This value is only relevant if several objects have been grouped under the Clone Object or if different shapes are to be cloned. This value determines with which of the objects the sequence of clones along the assigned direction should begin.
  • Direction: +X, -X, +Y, -Y, +Z, or -Z – Here you specify the direction in which the clones are to be lined up or stacked. The direction specification refers to the axis of the BCloner object.

Gap: The placement of the clones takes their maximum dimensions into account. Use this value to add additional space between the clones or to manually reduce the space between the clones.

Aim Along (Z)

  • Match Plane (by default): The clones are rotated according to the axis directions of the virtual rectangular surface.
  • X, Y, Z: The clones are aligned along the X, Y, Z axis direction of the virtual rectangular surface.
  • Random: The clones are randomly aligned along one of the axis directions of the virtual rectangular surface.

Stack Distribution - Aim Along

Orientation

  • Forward (by default): The positive Z-axis of the clone objects points in the direction selected under Align along the virtual rectangular surface. The sign of the Direction setting also plays a role here.
  • Backward: The positive Z-axis of the clone objects points in the opposite direction to the one selected under Align along the virtual rectangular surface. The sign of the Direction setting also plays a role here.
  • Forward And Backward: The positive Z-axis of the clones points randomly in the positive or negative axis direction of the virtual plane selected under Align Along.

Stack Distribution - Orientation

Spin & Scale Axis

  • Spin:
    • Auto (by default): All clones are rotated identically around their orientation axis.
    • Opposite: The clones are rotated by 180° around their alignment axis. In the case of airplanes, they would now fly lying on their backs.
    • Random: The rotation takes place randomly in 90° steps around the alignment axis.
  • Scale Axis:
    • Direction: The clones can be scaled along their Y-axis. The size of the clones remains unchanged in the perpendicular axis directions.
    • Uniform: The clones are proportionally enlarged or reduced in size.

Stack Distribution

Random Scale & Stretch to Fill

  • Random Scale: The strength of a random scaling of the cloned objects can be set here. Under Scaling Axis, you can select whether these changes in size should only take place in the direction of the line-up or proportionally. Like all random effects, this calculation is also based on the Seed value. A change in the Seed value therefore also leads to a recalculation of the scaling variations.
  • Stretch to Fill: Stretches the Scale of Clones to Fill the gaps completely.

Pivot & Seed

  • Pivot XYZ: These three percentage values refer to the bounding box, which encloses all cloned objects. The position of the cloned objects can thus be changed within this bounding box, e.g., to display books standing on a board, although the books have different sizes.
  • Seed: This distribution offers random modes based on this value when selecting the objects to be cloned.
