Partition Distribution

This distribution can be loaded directly using the Advanced Distribution button in the BCloners Extension Panel or place it manually if you change the Distribution Menu to Advanced and set the Distribution Object to the parameter Distribution Object. This distribution is based on the Partition Modifier and its subdivision of a rectangular area. The resulting individual surfaces are used to place and scale the cloned objects.

Partition Distribution

Basic Parameters

Direction:

Choose the Axis that is as perpendicular direction of virtual rectangular.

  • +X, -X, +Y, -Y, +Z or -Z

Size Width:

Side length of the plane in the X direction.

Size Height:

Side length of the plane in the Y direction.

Gap and Orientation

Gap:

The virtual rectangular area is divided horizontally and vertically several times, creating rectangular areas of different sizes, which are then evaluated for the distribution of the clones. The Gap value can be used to proportionally reduce the size of these areas within the virtual rectangular area. This creates empty gaps between the clone areas. In relation to the clones, larger Gap values mean that the cloned objects can no longer touch each other.

Partition Gap

Aim Along (Z)

Aim Along (Z): Match Plane (by default)

The clones are rotated according to the axis directions of the virtual rectangular surface.

Aim Along (Z): X, Y, Z

The clones are aligned along the X, Y, Z axis direction of the virtual rectangular surface.

Aim Along (Z): Random

The clones are randomly aligned along one of the axis directions of the virtual rectangular surface.

Orientation

Partition Orientation

Orientation: Forward (by default)

The positive Z-axis of the clone objects points in the direction selected under Aim Along the virtual rectangular surface. The sign of the Direction setting also plays a role here.

Orientation: Backward

The positive Z-axis of the clone objects points in the opposite direction to the one selected under Aim Along the virtual rectangular surface. The sign of the Direction setting also plays a role here.

Orientation: Forward And Backward

The positive Z-axis of the clones points randomly in the positive or negative axis direction of the virtual plane selected under Aim Along.

Spin

Partition Spin

Spin: Auto (by default)

All clones are rotated identically around their orientation axis.

Spin: Opposite

The clones are rotated by 180° around their alignment axis. In the case of our airplanes, they would now fly lying on their backs.

Spin: Random

The rotation takes place randomly in 90° steps around the alignment axis.

Cut Algorithm & Subdivision Types

Partition Cut Algorithm

Cut Algorithm: Simple (by default)

The cut is made regardless of the aspect ratio of the virtual rectangular area and any areas already created there by previous cuts.

Cut Algorithm: Largest

Largest Edge (Stable Geo Dependent).

Subdivision Types

Subdiv Type: Single (by default)

The subdivisions are added individually and their number can be set via the Cuts value.

Subdiv Type: Double

The subdivisions are added to all surfaces at the same time, alternating in the horizontal and vertical directions. The number of these subdivisions is controlled via the Cut Iterations value.

Subdiv Type: Quad

The subdivisions are added to all surfaces simultaneously in the horizontal and vertical directions. The number of these subdivisions is controlled via the Cut Iterations value. Please note that, in contrast to Double mode, even lower Cut Iterations can lead to finer subdivisions. Therefore, only increase this value cautiously to avoid creating too many or too small clones.

Partition Distribution - Subdivision Types

Subdivision Controls

Cuts:

Here you set the number of sections to be added for the Single Subdivision Type.

Cut Iterations:

Here you set the sections to be added for the Subdivision Type modes Double and Quad. As these sections are calculated for all surfaces, many correspondingly small surfaces can be created very quickly. Therefore, increase this value carefully, especially in the Quad setting.

Randomization

Partition Distribution - Randomize Subdivision

Randomize Density:

By increasing this value, the areas in which subdivisions are added can be more unevenly distributed. For large percentage values, areas can remain completely without new subdivisions, while other areas are heavily subdivided. Please note that any change to this value leads to a recalculation of all sections.

Partition Distribution - Randomize Subdivision

Randomize Cuts:

Increasing this value moves the edges within the original rectangular area. An originally regularly divided area can thus be subdivided into more unevenly scaled areas. Please note that any change to this value will result in a recalculation of the sections.

Cut Orientation

Cut Orientation: Manual

The direction of the subdividing cuts can be specified manually via Orientation H/W. Depending on the percentage value defined there, the cuts are then added primarily in the horizontal or vertical direction. With an Orientation H/W value of 50%, an attempt is made to balance the ratio between horizontal and vertical sections.

Cut Orientation: Alternating (by default)

The direction of the subdivision cuts alternates between horizontal and vertical for each new cut.

Cut Orientation: Longest Edge

Longest Edge (Stable Geo Dependent).

Orientation H/W:

If Manual Cut Orientation is selected, you can use this value to influence the main direction for the subdivision cuts. At 50%, a balanced choice of cutting paths is achieved, half of which run horizontally and half vertically.

Scaling & Matching

Match Scale To: None

The size of the cloned objects remains unchanged.

Match Scale To: Width/Height

The X size of the cloned objects adapts to the corresponding width/height of the subdivision area on the virtual rectangle.

Match Scale To: Constrained

The scaling is always based on the shortest edge of the respective subdivision area within the virtual rectangle and scales the clones proportionally to this.

Match Scale To: Scale To Fit (by default)

The X and Y sizes of the clones are scaled identically to the available space in the corresponding area of the virtual rectangle. This inevitably leads to a distortion of the clones.

Scale Facing

Scale Facing: None

The clones can be scaled freely, percentage based on their original size in this direction via the Scale Facing value.

Scale Facing: Set

The size of the clones in this direction can be specified in absolute terms using the Scale Set value.

Scale Facing: Proportional (by default)

The size of the clones in this direction is based on the scaling in the other axis directions, but can also be corrected as a percentage using the Scale value.

Scale Facing: Random

The clones are scaled randomly in the corresponding direction. However, with Random Scale Min and Random Scale Max you can define percentage maximums that refer to the original dimensions of the cloned objects in this direction.

Scale Parameters

Scale:

This value refers to the original dimension of the cloned objects in the Z direction of the distribution plane and can be used in various Scale Facing modes to adjust the object sizes.

Scale Set:

For Scale Facing Set, an absolute dimension of the clones in the direction of the Z-axis at the distribution plane can be specified here.

Min / Max:

  • Min: Adjust the Minimum Random Scale.
  • Max: Adjust the Maximum Random Scale.

Pivot XYZ:

These values describe the position of the starting point for the scaling within the bounding box of the clones. With the default values 0%, the reference system is exactly in the middle of the bounding boxes. Scaling then takes place uniformly in all axial directions of the clones.

Advanced Controls

Cut Cap:

This value can be used to define an upper limit for the number of sections and subdivisions to be calculated. This can, for example, prevent accidental overloading of the working memory or freezing if the subdivision levels are set too high. If you require a large number of subdivisions, it may also be necessary to increase this value.

Seed:

Changes the Seed of the Random parameters.

Show Plane:

Shows the Original Plane that is used to generate the Distribution.

Animation

Random Offset:

Here you can define the maximum displacement of the individual sections and subdivisions during the animation. The actual freedom of movement of the subdivisions depends on the length of the respective edge that is being subdivided.

Noise Frequency:

Here you control the speed at which the subdivisions are moved. Higher values lead to faster changes.

Animation Speed:

This controls the overall speed of the animation. The resulting movement speed results from the multiplication of Animation Speed and Noise Frequency. If one of these values is 0, no more animation takes place.

Loop Period:

Here you can define a number of frames after which the animation should repeat automatically. With the default value 0, there is no repetition and the animation is purely random, regardless of how long it is viewed.
