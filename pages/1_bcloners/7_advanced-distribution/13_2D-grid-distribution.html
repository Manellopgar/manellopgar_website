Projected 2D Grid Distribution

This distribution can be loaded directly using the Advanced Distribution button in the BCloners Extension Panel or placed manually if you change the Distribution Menu to Advanced and set the Distribution Object to the parameter Distribution Object. All Clones are placed in a regular grid, which is then projected onto the surface of an assigned object. The copies can also be aligned relative to this surface.

The object to be projected onto must be placed on the Base Object parameter.

Projection animation on a Terrain surface

Base Object & Distribution Parameters

Base Object: Drag Your Custom Asset (Base Object). For example: Terrain, Ground... etc.
Position Strength: Defines the amount of Strength for the Clones Project Position on the Base Object.
  • • Clones: Iterate: The distribution automatically runs through all objects in sequence from top to bottom, as they were grouped under the Cloner object. After placing the last object, this sequence starts again with the topmost object under the Cloner object.
  • • Clones: Random: The object currently to be cloned is selected at random from the objects that have been grouped to the clone object.
ID Asset: Offset the index of each Asset.

Grid Settings & Mode

Count X: Number of clones in the X direction.
Count Y: Number of clones in the Y direction.
  • • Mode: Per Step: When selected, the values will reflect the difference to the previous or next clone.
  • • Mode: End Point: When selected, the values will reflect the difference between the first and last clone, and the values in-between will be interpolated.
Size XY: Here you specify the dimensions of the virtual rectangular area on which the clones are placed.
Plane: This setting defines the orientation of the rectangular area in which the copies are originally created.

Projection Methods

  • • Projection: Direction: Starting from the regular grid positions of the clones, calculation rays are sent out parallel to the vertical on this grid level. Where these collide with the object assigned as the projection target, a new position is calculated.
  • • Projection: Nearest: Here, too, calculation rays are sent out from the original grid positions in the direction of the child object in the distribution, but these do not necessarily run parallel to the vertical on the grid surface; instead, the rays determine the closest position on the subordinate grouped object for each clone.

Projection Schematic (Reference)

Plane Projection Directions

  • • Plane Projection: Forward: Ensures collision rays that run parallel to the positive Y axis. The object that is projected onto should therefore also lie above the XZ plane of the Cloner object.
  • • Plane Projection: Backward: The calculation rays would run exactly in the opposite direction, i.e., vertically downwards.

Plane Projection Direction Examples (Reference)

Alignment & Smoothing

Shift Position: This can be used to adjust the determined position of the object copies. The percentage value refers to the dimensions of the bounding box of the cloned objects. Depending on whether Align to Normal is used or not, the clones can be moved in the direction of the projection direction or relative to the surface normal.

Shift Position Adjustment (Reference)

  • • Align To Normal: The separate Smooth option can be added to a smoothing interpolation of neighboring normal directions.
  • • Smooth (True Normal): If Align to Normal is active, the directions of neighboring normals can also be interpolated with this option in order to smooth the transitions between the normals.

Align to Normal and Smooth Interpolation (Reference)

Scales & Randomization

  • • Scale Axis: Orientation: The clones can be scaled along their Y-axis. The size of the clones remains unchanged in the perpendicular axis directions.
  • • Scale Axis: Uniform: The clones are proportionally enlarged or reduced in size.

Scale Axis Options (Reference)

  • • Random Scale: This percentage value refers to the size of the bounding box on the cloned objects and indicates their maximum size variation. With a value of 100%, the clones can therefore become a maximum of twice as large.
Seed: Changes the Seed of the Random parameters.
