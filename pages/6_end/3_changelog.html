Changelog 1.5.0 - 26-06-24

Note: B4D Tools 1.5.0 is NOT COMPATIBLE with previous versions. The new BCloners distributions and the fixed Min/Max parameters on Effectors involve significant changes that do not work properly with old setups.

NEW TOOLS!

In the BCloner Panel, you will find the new Add Advanced Distribution option:

  • Advanced Distributions: This option allows you to choose between 6 advanced distribution arrays for any BCloner, plus one empty slot for creating your own custom advanced distribution. All options include bounding box calculations to simplify the setup of any kind of instance: 2D Grid Distribution, Cannonball Distribution, Smart Hex Distribution, Spline Bed Distribution, and Stack Distribution.

New Fields

  • B4D Field Set VERTEX Weight: This field allows you to use any Vertex Weight from a Custom Object as a control field for any B4D Tool.

For example, this makes Dynamic Paint compatible with B4D Tools, providing greater flexibility. The bug preventing height map functionality has been fixed, and it now works as expected.

New Object Tools

  • Object BMograph ANIMATOR: This tool is highly useful and features a vibrate tool similar to C4D's. Additional options include ping-pong and repeat animation modes. All tools are compatible with single objects as well as any BCloner or instance objects. The Selection option has been fixed to work as expected.
  • Object Camera Tools: This category now includes two tools. Frustum Culling is available, allowing you to render only instances or geometry visible in the active camera. You can adjust the margin via the Margin parameter or use a Custom Camera for added flexibility. The Level Of Detail option allows you to use lighter assets for instances located at a distance from the Active Camera.

New Material Tools

  • Images as MatSlots Button: Opens a file browser to select a folder containing images or videos. You can assign each file to a clone of the selected BCloner. This tool adds one material per file, preconfigured with two node groups: one for UV map setup (allowing quick changes across all materials) and another for Shader Node configuration. The filename is automatically used as the material name.
  • Images as Slots: If the MultiShader modifier is not present, it is now added automatically.
  • Video Import Options: When importing video files, you now have specific options to configure useful settings for all videos, saving significant time.
  • Override MultiShader Tool: Allows you to override MultiShader MatSlots from any type of BCloner. This is handy for keeping one animated BCloner while creating multiple variants with different textures but the same animation.
  • Transformation Menu: Added to allow selection between Original and Relative Origin modes.
  • Advanced Options: The parameter As Instances is now enabled by default, as it is required for most cases.

INTERFACE UPDATES

I have refactored the full UI of B4D Tools to streamline future updates, keep the code clean, and maximize code reusability. You will notice a slightly different interface where panel order can be customized. All features from the previous version are available, along with several improvements.

New Website Documentation

The full documentation is now fully text-based and accessible via the addon's preferences panel. The main menu follows the same order as the addon panels. This is the first version, so it will be improved in the coming weeks. Your feedback is welcome.

  • Navigaion: Use the main menu to browse sections. Toggle between Dark and Light mode using the button in the top-right corner.
  • Search: Use the search field in the top-right corner. Results are highlighted after a short processing delay.
  • Initial Guide: Check the "Initial Guide" pages for critical explanations on how the addon works with modifier stacks and the Outliner, along with recommendations for the best experience.

Viewport Feedback & Usability

  • Collection Wireframes: If the Collection option is active and the selected collection is empty, B4D Tools now displays wireframe cubes in the viewport to provide visual feedback on how assets will be placed.
  • Effector Sound & Field Sound: The button now has two states: one for automatic sound baking to the control empty, and another to restore the standard Blender UI layout. This prevents unwanted changes to your workspace.
  • Incompatible Object Detection: If incompatible objects are selected when adding a tool, an error message now appears listing the compatible objects, preventing previous errors.
  • BTracer Icon: The icon has been updated to distinguish it from the Animator Tools.

Naming Conventions

  • Advanced Panels: All Advanced Parameter/Setting panels are now named "Advanced Parameters" for cohesive naming.
  • Children/Assets: "Separated Children" and "Reset Children" are now named "Separated Assets" and "Reset Coordinates" to align with the rest of the addon's terminology.

AUTOMATIC DETECTION Function

This new function simplifies setup by automatically detecting extra selected objects and assigning them to the appropriate parameters:

  • BCloner Object Mode: Extra selected object → Custom Mesh parameter.
  • BCloner Spline Mode: Extra selected curve → Spline/Curve parameter.
  • Effector Spline: Extra selected curve → Spline/Curve parameter.
  • Effector Target: Extra selected object → Target parameter.
  • Effector Inheritance: Extra selected object → Target parameter.
  • Effector Volume: Extra selected object → Volume Object parameter.
  • Field Object Proximity: Extra selected object → Object Proximity parameter.
  • Field Spline Object: Extra selected object → Spline Object parameter.
  • BMoText Object: Extra selected curve → Curve/Spline Path parameter (and automatically enables Text On Curve).
  • BTracer: Extra selected object → Link Tracer parameter.
  • BTracer To Mesh Object: Extra selected curve → Profile Curve parameter.
  • Override Imgs as Slots: If two objects are selected, the second object is automatically added to the "Original BCloner" socket, while the active object receives the BCloner and Multishader setup.

All BCloners

  • Asset Type Parameter: Replaces "Use Collection" with three options: Self (uses original mesh as instance), Object (choose a custom object), and Collection (same as previous Use Collection). This clarifies setup when using a single custom object.
  • Clones from Distribution: Now a button switch. This dynamic interface hides unnecessary parameters, showing only what is relevant to the active menu.
  • ID Asset & Random Seed: Fixed correct display for BCloner Grid Array, Linear, Radial, and Honeycomb modes.

BCloner Linear Mode

  • Min/Max Origin Pos: Added to change the origin of the Linear clones' start position. Default is 1 to maintain compatibility with previous version results.

BCloner Object Mode

  • Align Parameter: Fixed to use surface normals by default in both Surface and Volume modes.
  • Asset Type: Changed from Use Collection to Asset Type. Use Object for single assets or Collection for multiple assets. "Use Self" is unavailable due to Geometry Nodes limitations.

BCloner Radial Mode

  • Scale X and Y: Added to deform the original radial shape into an oval or similar form, providing more creative options.

BCloner Transform

  • Use Material & Material Selection: Both fixed to work as expected.

BCloner Advanced Distributions

  • Hidden Container: The object container for selected Advanced Distributions is now hidden in both Viewport and Render. You can still edit its parameters by selecting it in the Outliner.
  • PSR Follow: Fixed Rotation and Scale. The Advanced Distribution clones now correctly follow the BCloner's PSR (Position, Scale, Rotation) changes.

Advanced Projected 2D Distribution

  • Align To Normal: Fixed to work as expected.
  • Self Transformation: Fixed so that moving the BCloner correctly moves the Advanced Distribution accordingly.

Advanced Spline Bed Distribution

  • Scale Rail: Fixed to work as expected.

Advanced Stack Distribution

  • Random XYZ on Direction: Added to provide more control over randomizing the size of all clones.

Effectors Refactoring (Plain, Random, Step, Image Text, Noise Tex, Sound, Volume, Time, Formula)

The Min/Max parameters have been refactored, breaking compatibility with previous versions. Default scale values are now 0 (previously 1) to ensure correct functionality. Visibility parameter is also fixed.

Effector Step

  • Spline Nodegroup: Added a Float Curve to control step interpolation calculation, offering flexibility similar to C4D. Note: Edit the curve in the Geometry Nodes editor for now; interface control will be added in the future.
  • Scale X Axis: Fixed to work as expected.

Effector Volume

  • Volume Selection: Fixed to work as expected.

Effector Delay

Complete refactor to ensure Position, Rotation, and Scale work correctly and can be combined without issues.

  • String Mode: Added Stiffness and Damping parameters to control the physics of the String mode. These are located in Advanced Options.
  • Mode Options: Supports both Even and String modes.

Effector Noise Tex

  • Noise Seed: Added to allow manual seed changes when Auto Animation is disabled.
  • Noise Type Menu: Added to select the desired noise type.
  • Auto Animation Menu: Now contains Animation Speed (set to 100% default) and Loop Animation.
  • Loop Animation: New parameter to enable noise looping. When enabled, the Frame Loop Out parameter appears, allowing you to define the end frame for the loop.

Effector Random

  • Animation Speed: Changed to a percentage-based parameter with a default value of 100.
  • Loop Animation: A new toggle to enable noise loop animations. When enabled, the Frame Loop Out parameter becomes visible, allowing you to define the specific frame where the loop concludes. This simplifies the creation of seamless noise loops for clones.
  • Auto Animation: When enabled, this reveals the following controls:
  • Animation Speed: Controls the playback speed.
  • Loop Animation: Toggles the looping behavior.
  • Use Random Time: Refactored into a dedicated menu for easier access.
  • Rand Time Seed: Visible when Random Time is enabled, allowing you to adjust the seed value for varied noise patterns.
  • Min/Max Refinement: The Min and Max parameters have been refactored to work properly, providing precise control over transformations applied by the Effector Random.
  • Scale Parameter Fix: Corrected a bug where the Scale parameter did not interact correctly with Field tools. It now functions as expected across all contexts.

Effector Formula

  • Sine Axis: Added Sine Axis menu, allowing you to choose which axis to use for the Sine Animation.
  • Animation Speed: Changed to a percentage parameter with a default value of 100.
  • Loop Animation: New parameter that can be enabled to create a noise loop animation. When enabled, the Frame Loop Out parameter appears, allowing you to define the frame at which the loop ends, making noise loop animations for clones straightforward.
  • Wave/Voronoi Type: Renamed the Wave/Voronoid Texture Configuration menu to Wave/Voronoi Type for a more compact label.
  • Auto Animation: When enabled, displays the Animation Speed and Loop Animation parameters.

Effector Noise Texture

  • UV Map Fix: Fixed the UV Map for the Noise, ensuring it now works as expected.

Effector Spline

  • Tooltips: Added tooltips to the Transform Mode menu for better guidance.

Field Random

  • Noise Seed: Added a Noise Seed parameter when Auto Animation is disabled, allowing users to change the noise seed.
  • Animation Speed: Changed to a percentage parameter with a default value of 100.
  • Loop Animation: New parameter that can be enabled to create a noise loop animation. When enabled, the Frame Loop Out parameter appears, allowing you to define the frame at which the loop ends, making noise loop animations for clones straightforward.
  • Auto Animation: When enabled, displays the Animation Speed and Loop Animation parameters.
  • Round Falloff: Added Round Falloff option, useful when using the random result as a Custom Field Selection.

Field Formula

  • Sine Axis: Added Sine Axis menu, allowing you to choose which axis to use for the Sine Animation.
  • Animation Speed: Changed to a percentage parameter with a default value of 100.
  • Loop Animation: New parameter that can be enabled to create a noise loop animation. When enabled, the Frame Loop Out parameter appears, allowing you to define the frame at which the loop ends, making noise loop animations for clones straightforward.
  • Wave/Voronoi Type: Renamed the Wave/Voronoid Texture Configuration menu to Wave/Voronoi Type for a more compact label.
  • Auto Animation: When enabled, displays the Animation Speed and Loop Animation parameters.

Multishader

  • Performance Improvement: Improved performance by preventing unnecessary duplicated geometry generation.
  • Keep Mesh (Not Instances): Added a parameter to keep geometry that is not an Instance.
  • Island Index Splitting: Added the option to use the Island Index to split to instances, useful when working with multiple Multishaders.

Mapping Color to Instances

  • Mode Menu: Replaced the old Boolean switch for ColorToVertex/ColorToInstances with a new Mode menu for clearer selection.
  • Fix Texture: Added a third option called Fix Texture, functioning similarly to the Fix Texture parameter in C4D Clones. This allows you to apply any texture and fix it onto the clones, projecting the texture on top.

BTracer

  • Asset Type: Changed "Use Collection" to Asset Type to align with the new naming conventions of BCloners.
  • Split To Instances: Added a parameter to split geometry, useful when merged geometry is added to the Trace Link Object.
  • Split To Instances Selection: Added the ability to set a custom selection for splitting to instances.

BEmitter

  • Asset Type: Changed "Use Collection" to Asset Type to align with the new naming conventions of BCloners.
  • Empty Controller: Added the Empty Controller as a separate object, providing rotation control when the emitter is emitting particles/instances.

Matrix ToInstances

  • Asset Type Options: Changed the "Use Collection" parameter to Asset Type with three options: Self (uses the original mesh as instances, default), Object (allows choosing a custom object as an instance), and Collection (same as the previous Use Collection). This clarifies the process when only a custom object is needed.

BMograph Selection

  • Even Indices Selection: Added the ability to select even indices of clones, selecting only the even-numbered indices.
  • Display Index Fix: Fixed the Display Index to prevent duplicating original instances.
