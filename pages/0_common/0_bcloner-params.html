Distribution Type

This first parameter allows us to choose the Distribution Type with 2 options: Basic and Advanced. By default, the Basic mode is enabled. For more information about Advanced Distributions, please check the Advanced Distributions in Main Menu.

Distribution Type: Basic

This option is the classic option of BCloners; you will see all the classic options to control and generate the clones.

Distribution Type: Advanced

This option enables the Advanced Distribution parameters. Here, the BCloner uses an extra object that contains the Advanced Matrix distribution modifier. In this option, you can control and change it.

Clones from Distribution: Enable

Uses the Clones assigned in the Distribution Object.

Asset Type

This menu allows us to choose the input Asset Type. You can choose between Self, Object, and Collection. Except for BCloner Object, which only has Object and Collection.

Asset Type: Self

Uses the Self Object as an object for the clone.

Asset Type: Object

Uses the added Custom Object to Object parameter as a clone.

Object: Drag Your Custom Object that is used as Clone.

Asset Type: Collection

Uses the Objects of the added Collection to Collection parameter as clones. When this mode is selected, you will have more parameters to control the generated clones.

Collection: Drag Your Custom Collection with the Assets that are used as Clones.

BCloner - Asset Type (Used Grid Array as an example)

Clones Mode

Select how the assets from the collection are distributed:

Clone Mode: Iterate

All objects directly subordinate to the cloner object will be cloned one after the other, as they are arranged in the object manager, from top to bottom and then from the front again. You also can change the ID Asset parameter.

ID Asset

Offset the index of each Asset.

Clone Mode: Random

All objects directly subordinate to the cloner object will be used in a random distribution to generate the clones. A Seed value parameter will be displayed in each case, which can be used to vary the random distribution. You also can change the Random Seed parameter.

Random Seed

Randomize the Seed of Clones Mode Random.

Clone Mode: Blend

You can blend between two objects with different shapes. This blend mode ensures proper interpolation between two objects with the same amount of vertices; this only works between the first and the last object of the collection. You must add any Effector with Other Parameters added after to work properly. More details in the Other Parameters of Effectors.

Clone Mode: Sort

The Sort mode will only have an effect when using Effectors (for those that support this, the Modify Clone parameter in the Parameters tab must be greater than 0). The corresponding Effector can then vary the position, size, angle, etc. of the clones. More details in the Other Parameters of Effectors.

BCloner - Clones Mode (Used Grid Array as an example)

Separate Assets

Allows the use of each child of the collection as a separate Clone. Only works if you have more than 1 Asset in the Custom Collection (Drop Your Assets Here).

Reset Coordinates

Reset Position, Rotation and Scale to BCloner Origin.

BCloner - Separate Assets (Used Grid Array as an example)

Instance Mode

This menu allows us to select the Viewport Instance Mode to improve performance.

Instance Mode: Instance

The clones are fully-fledged duplicates of the objects to be duplicated. They consume the same amount of memory as if each clone existed as a separate object.

Instance Mode: Render Instances

Realize all Instances to Real Geometry.

Instance Mode: Multi-Instances

This mode is particularly suitable for all types of vegetation. When it comes to cloning tens of thousands of trees or tufts of grass, for example, this mode is a must. This mode enables the Viewport Mode.

Viewport Mode: Off

No instances will be displayed.

Viewport Mode: Points

The instance positions will be displayed with small dots.

Viewport Mode: Matrix

The instance positions will be displayed with small squares (as usual for the matrix object).

Viewport Mode: Bounding Box

The instances will be displayed with their bounding box. This allows the approximate clone sizes to be estimated.

Viewport Mode: Object

The instances will be displayed in full.
