Distribution Type

This parameter allows us to choose the Distribution Type with two options: Basic and Advanced. By default, Basic is enabled.

  • Distribution Type: Basic
    The classic option of BCloners. You will see all the classic options to control and generate clones.
  • Distribution Type: Advanced
    Enables the Advanced Distribution parameters. The BCloner uses an extra object that contains the Advanced Matrix distribution modifier. In this option, you can control and change it. For more info, please check Advanced Distributions.
  • Clones from Distribution: Enable
    Use the Clones assigned in the Distribution Object.

Asset Type

This menu allows us to choose the input Asset Type. You can choose between Self, Object, and Collection. Note that the BCloner Object only supports Object and Collection.

Asset Type: Self

Uses the Self Object as an object for the clone.

Asset Type: Object

  • Add Custom Object: Drag your custom object to use it as a clone (Object parameter).

Asset Type: Collection

Uses the objects in the added collection as clones. When this mode is selected, more parameters appear for controlling the generated clones.

  • Add Custom Collection: Drag your custom collection with assets to use as clones (Collection parameter).

Clones Modes

Select how the assets from the collection are distributed:

  • Iterate: All objects directly subordinate to the cloner object will be cloned one after another, as they are arranged in the object manager (top to bottom, front again). You can also change the ID Asset parameter.

  • ID Asset: Offset the index of each asset.
  • Random: All objects directly subordinate to the cloner object will be used in a random distribution to generate clones. A Seed value parameter will be displayed, which can vary the random distribution.
  • Random Seed: Randomize the seed of Clones Mode Random.
  • Blend: You can blend between two objects with different shapes. This mode ensures proper interpolation between two objects with the same amount of vertices. It only works between the first and last object of the Drop Your Assets Collection.
  • Sort: The Sort mode has an effect when using Effectors (for those that support this, the Modify Clone parameter in the Parameters tab must be greater than 0). The corresponding Effector can then vary the position, size, angle, etc. of the clones.
  • Separate Assets: Allows the use of each child of the collection as a separate clone. Only works if you have more than 1 asset in the Custom Collection.
  • Reset Coordinates: Reset Position, Rotation, and Scale to BCloner Origin.

Instance Mode

This menu allows us to select the Viewport Instance Mode to improve performance.

  • Instance: The clones are fully-fledged duplicates of the objects to be duplicated. They consume the same amount of memory as if each clone existed as a separate object.
  • Render Instances: Realize all instances to real geometry (for rendering only).
  • Multi-Instances: Particularly suitable for vegetation (e.g., thousands of trees or grass). This mode enables the Viewport Mode options below.

Viewport Modes

Select how instances are displayed in the viewport when using Multi-Instance mode:

  • Off: No instances will be displayed.
  • Points: The instance positions will be displayed with small dots.
  • Matrix: The instance positions will be displayed with small squares (as usual for the matrix object).
  • Bounding Box: The instances will be displayed with their bounding box, allowing approximate clone sizes to be estimated.
  • Object: The instances will be displayed in full.
