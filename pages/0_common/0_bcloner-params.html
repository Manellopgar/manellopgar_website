Distribution Type

This first parameter allows us to choose the Distribution Type with 2 options: Basic and Advanced. By default, Basic is enabled. For more information about Advanced Distributions, please check the Advanced Distributions.

  • Basic: This option is the classic mode of BCloners. You will see all classic options to control and generate the clones.
  • Advanced: This option enables the Advanced Distribution parameters. Here, the BCloner uses an extra object that contains the Advanced Matrix distribution modifier. In this option, you can control and change the distribution.

Clones from Distribution: Uses the Clones assigned in the Distribution Object.

Asset Type

This menu allows us to choose the input Asset Type. You can choose between Self, Object, and Collection. Note that BCloner Object only has Object and Collection options.

  • Self: Uses the Self Object as the object for the clone.
  • Object: Uses the added Custom Object from the Object parameter as the clone.

Object: Drag Your Custom Object that is used as Clone.

  • Collection: Uses the Objects from the added Collection (Collection parameter) as clones. When this mode is selected, more parameters become available to control the generated clones.

Collection: Drag Your Custom Collection with the Assets that are used as Clones.

Clones Mode

Select how the assets from the collection are distributed:

  • Iterate: All objects directly subordinate to the cloner object will be cloned one after the other, as they are arranged in the object manager, from top to bottom and then from the front again. You can also change the ID Asset parameter.

ID Asset: Offset the index of each Asset.

Object: Uses the added Custom Object from the Object parameter as clone.

  • Random: All objects directly subordinate to the cloner object will be used in a random distribution to generate the clones. A Seed value parameter will be displayed in each case, which can be used to vary the random distribution. You can also change the Random Seed parameter.

Random Seed: Randomize the Seed of Clones Mode Random.

  • Blend: You can blend between two objects with different shapes. This blend mode ensures proper interpolation between two objects with the same amount of vertices; this only works between the first and the last object of the collection. You must have any Effector with Other Parameters added to work properly. More details in the Other Parameters of Effectors.
  • Sort: The Sort mode will only have an effect when using Effectors (for those that support this, the Modify Clone parameter in the Parameters tab must be greater than 0). The corresponding Effector can then vary the position, size, angle, etc. of the clones. More details in the Other Parameters of Effectors.

Separate Assets: Allows the use of each child of the collection as a separate Clone. Only works if you have more than 1 Asset in the Custom Collection (Drop Your Assets Here).

Reset Coordinates: Reset Position, Rotation and Scale to BCloner Origin.

Instance Mode

This menu allows us to select the Viewport Instance Mode to improve performance.

  • Instance: The clones are fully-fledged duplicates of the objects to be duplicated. They consume the same amount of memory as if each clone existed as a separate object.
  • Render Instances: Realize all Instances to Real Geometry.
  • Multi-Instances: This mode is particularly suitable for all types of vegetation. When it comes to cloning tens of thousands of trees or tufts of grass, for example, this mode is a must. This mode enables the Viewport Mode.
  • Off: No instances will be displayed.
  • Points: The instance positions will be displayed with small dots.
  • Matrix: The instance positions will be displayed with small squares (as usual for the matrix object).
  • Bounding Box: The instances will be displayed with their bounding box. This allows the approximate clone sizes to be estimated.
  • Object: The instances will be displayed in full.
